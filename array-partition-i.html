Array Partition I

Given an array of 2n integers, your task is to group these integers into n pairs of integer, say (a1, b1), (a2, b2), ..., (an, bn) which makes sum of min(ai, bi) for all i from 1 to n as large as possible.

  • n is a positive integer, which is in the range of [1, 10000].
  • All the integers in the array will be in the range of [-10000, 10000].

Example:

  • Input: [1,4,3,2]
  • Output: 4
  • Explanation: n is 2, and the maximum sum of pairs is 4.

Python Solution:

class Solution(object):
    def arrayPairSum(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        return sum(sorted(nums)[::2])
  • Time complexity is O(NlgN) because of sort.
  • Space complexity is O(N), because sorted returns a new list.
  • We could use nums.sort() to sort nums in-place, but discuss with your interviewer first.

C Solution:

int arrayPairSum(int* nums, int numsSize) {
    int count[20001] = {0};
    int i;
    for (i = 0; i < numsSize; i++) count[nums[i] + 10000]++;
    int res = 0, carry = 0;
    for (i = 0; i < 20001; i++) {
        if (!count[i]) continue;
        count[i] -= carry;
        carry = 0;
        if (!count[i]) continue;
        res += (i - 10000) * ((count[i] + 1) >> 1);
        carry = count[i] & 1;
    }
    return res;
}
  • Time complexity is O(N).
  • Space complexity is O(N).
    • count[20001] can be described as O(20001), which is same as O(1).
    • but 1 <= N <= 10000, taking this into account, the space complexity should be O(N).
