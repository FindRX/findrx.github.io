Function Decorators and Closures

A Decorator is a callable that takes another function as argument (the decorated function) and returns another callable object.

Decorators are just syntactic sugar, and they are executed immediately when a module is loaded.

Most decorators define an inner function and return it to replace the decorated function.

  • decorators can be used many times, so each time the inner function is a new object created when the decorators run.

A closure is a function with an extended scope that encompasses nonglobal variables referenced in the body of the function but not defined there.

#obj is a function.
obj.__code__.co_varnames  # names of local variables.
obj.__code__.co_freevars  # names of free variables.
obj.__closure__ # the bindings of the variables in co_freevars.
obj.__closure__[0].cell_contents # the actual value of the first freevar.

Python has three built-in functions that are designed to decorate methods: property, classmethod and staticmethod.

How do you make a decorator accept other arguments?

  • make a decorator factory that takes those arguments and returns a decorator, which is then applied to the function to be decorated.
  • So, just another level.
  • Parameterized decorators almost aways involve at least two nested functions.
