Scopes in Python3

In all cases, the scope of a variable (where it can be used) is always determined by where it is assigned in your source code and has nothing to do with which funtions call which.

Python assumes that a variable assigned in the body of a function is local.

  • When python compiles the body of the function, it decides that valude is local because it is assigned within the function.
  • The generated bytecode reflects this decision and will try to fetch the variable from the local enviroment.

If we want the interpreter to treat the variable as a variable in spite of the assignment within the function, we use the global declaration.

List comprehensions leak the loop control variable in Python 2 but not in Python 3.

Set and dict comprehensions don't leak in both Python 2 and Python 3.

Generators don't leak in both Python 2 and Python 3.
