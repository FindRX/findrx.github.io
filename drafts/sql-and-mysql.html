SQL and MySQL

Front

The model for a relational database states that

  1. A relational database consists of tables.
  2. Tables consist of rows/records and columns/fields.

Although SQL is a standard language, each vendor has its own SQL dialect, or variant, that may include extensions to the standards.

SQL is a freeform language. That means that you can include line breaks, spaces and indentation without affecting the way the database interprets the code.

In addition, SQL isn't case-sensitive like some languages. That means that you can use uppercase or lowercase letters or a combination of the two without affecting the way the database interprets the code.

In MySQL, a string can contain any combination of characters, and a function performs an operation and returns a value.

To code a literal value for a string, enclose one or more characters within single quotes ' or double quotes ".

To include a single quote within a literal value for a string, code two single quotes. Or use double quotes instead of single quotes to start and end the literal value.

How to use SQL from an application program

Most modern languages provide an API that allows you to connect to a MySQL database.

An API uses a piece of software known as a database driver to communicate with the database. For some languages, the database driver is built in. For others, you need to download and install a database driver.

For PHP, there are two APIs: mysqli (MySQL Improved Extension) API and PDO (PHP Data Objects) API. Both APIs use the same database driver which is typically included as part of the PHP language.

For Java, you typically use the JDBC (Java Database Connectivity) API. This API requires a driver to communicate with MySQL. In most cases, you can use the Connector/J driver that's available from the MySQL web site to connect a Java application to a MySQL database.

SQL Statements

The statements that work with the data in a database are called the data manipulation language (DML).

The statements that create databases and work with the objects within a database are called the data definition language (DDL).

On large systems, these statements are used exclusively by database administratiors (DBAs).

DML:

  1. SELECT
  2. INSERT
  3. UPDATE
  4. DELETE

DDL:

  1. CREATE DATABASE
  2. DROP DATABASE
  3. CREATE TABLE
  4. ALTER TABLE
  5. DROP TABLE
  6. CREATE INDEX
  7. ALTER INDEX
  8. DROP INDEX

Coding Recommendations:

  • Capitalize all keywords and use lowercase for the other code in a SQL statement.
  • Separate the words in names with underscores, as in invoice_number.
  • Start each clause on a new line.
  • Break long clause into multiple lines and indent continued lines.
  • Use comments only for portions of code that are different to understand. Then, make sure that the comments are correct and up-to-date.
    • To code a block comment, type /* at the start of the block and */ at the end.
    • To code a single-line comment, type -- followed by the comment.

Data Type

Numbers that don't include a decimal point are known as integer.

Numbers taht include a decimal point are known as real number.

The large object (LOB) data types are useful for storing images, sound, video, and large amounts of text.

You use CHAR type to store fixed-length strings. You use VARCHAR data type to store variable-length strings.

With MySQL 5.6 and later, the CHAR and VARCHAR types use the UTF-8 character set by default.

  • The character set uses up to three bytes to store each character.
  • When you use the UTF-8 character set with a CHAR tpe, MySQL must reserve three bytes for each character. As a result, MySQL must use 6 bytes for the CHAR(2) type, and it must use 30 bytes for the CHAR(10) type.
  • However, when you use the VARCHAR type, MySQL doesn't need to reserve space for each character. As a result, if you are using English letters and symbols, MySQL only uses one byte per character, plus one byte to store the length of the string.

With MySQL 5.5 and earlier, the CHAR and VARCHAR types used the latin1 character set by default.

You can use single or double quotes for string literals.

CHAR(M)

  • Fixed-length strings of character data where M is the number of characters, between 0 and 255.
  • With the UTF-8 character set, MySQL must reserve three bytes for each character in a CHAR.

VARCHAR(M)

  • Variable-length strings of character data where M is the maximum number of characters, betwwen 0 and 255.

The integer types:

  • BIGINT 8 bytes; INT 4 bytes; MEDIUMINT 3 bytes; SMALLINT 2 bytes, TINYINT 1 byte.
  • To specify a display size, such as INT(4), you can code it in parentheses after the data type.
    • But this only affects how MySQl displays the value, not how it stores the value.
  • INTEGER is a synonym for the INT type.
  • The BOOL and BOOLEAN types are synonyms for TINYINT(1).
    • To make that more intuitive, you can use the FALSE keyword, which is an alias for 0, and the TRUE keyword, which is an alias for 1.

DECIMAL(M, D)

  • Fixed-precision numbers where M specifies the maximum number of total digits (the precision) and D specifies the number of digits to the right of the decimal (the precision).
  • M can range from 0 to 65.
  • D can range from 0 to 30.
  • DEC, NUMERIC and FIXED types are synonyms for the DECIMAL type.

DOUBLE 8 bytes, FLOAT 4 bytes.

  • The REAL and DOUBLE PRECISION types are synonyms for the DOUBLE type.

Referential Integrity

The foreign key is simply one or more columns in a table that refer to a primary key in another table.

When MySQL enforces referential integrity, it makes sure that any changes to the data in the database don't create invalid relationships between tables.

When you use a inner join, rows from the two tables in the join are included in the result table only if their related columns match.

SELECT

The basic syntax for the SELECT clause (including five clauses.)

SELECT select_list
[FROM table_source]
[WHERE search_condition]
[ORDER BY order_by_list]
[LIMIT row_limit]

SELECT Clause

The SELECT clause is always the first clause in a SELECT statement.

The expanded syntax of the SELECT clause.

SELECT [ALL|DISTINCT]
       column_specification [[AS] result_column]
    [, column_specification [[AS] result_column] ...

To specify an alias for a column, use the AS phrase, although the AS keyword is optional.

  • To include spaces or special characters in an alias, enclose the alias in double quotes " or single quotes '.

Syntax of the WHERE clause

Syntax of the WHERE clause with comparison operators:

WHERE expression1 operator expression2
operators: =, <, >, <=, >=, <>, !=
  • To use a string literal or a date literal in a comprison, enclose it in quotes.
  • Literal values for dates must use this format: YYYY-MM-DD, this is the default date format used by MySQL.
  • To use a numeric literal, enter the number without quotes.
  • Character comparisons performed on MySQL databases are not case-sensitive.
  • If your compare a null value using comparison operators, the result is always a null value.

Syntax of the WHERE clause with logical operators:

WHERE [NOT] search_condition_1 {AND|OR} [NOT] search_condition_2 ...
  • When MySQL evaluates a compound condition, it evaluates the operators in this sequence: (1) NOT, (2) AND, (3) OR.

Syntax of the WHERE clause with an IN phrase

WHERE test_expression [NOT] IN ({subquery|expression_1 [, expression_2]...})

Syntax of the WHERE clause with a BETWEEN phrase

WHERE test_expression [NOT] BETWEEN begin_expression AND end_expression
  • The two expressions used in the BETWEEN phrase for the range of values are inclusive.

Syntax of the WHERE clause with a LIKE phrase

WHERE match_expression [NOT] LIKE pattern

Syntax of the WHERE clause with a REGEXP phrase

WHERE match_expression [NOT] REGEXP pattern
  • You use the LIKE and REGEXP operators to retrieve rows that match a string pattern, called a mark. Marks aren't case-sensitive.
  • Most LIKE and REGEXP phrases significantly degrade performance compared to other types of searches, so use them only when necessary.

ORDER BY can be used with column, alias, expression and column number.

JOIN AND UNION

A join lets you combine columns from two or more tables into a single result set.

Explicit syntax for an inner join:

SELECT select_list
FROM table_1
    [INNER] JOIN table_2 ON join_condition_1
    [INNER] JOIN table_3 ON join_condition_2
    ...
  • THE INNER keyword is optional and is seldom used.

A table alias is an alternative table name assigned in the FROM clause.

If you assign an alias to a table, you must use that alias to refer to the table throughout your query. You can't use the original table name.

A join condition can include two or more conditions connected by AND or OR operators.

A self-join joins a table to itself.

  • Although self-join are rare, they are sometimes useful for retrieving data that can't be retrieved any other way.

Implicit syntax for an inner join:

SELECT select_list
FROM table_1, table_2[, table_3] ...
WHERE table_1.column_name operator table_2.column_name
    [AND table_2.column_name operator table_3.column_name]...

Explicit syntax for an outer join:

SELECT select_list
FROM table_1
    {LEFT|RIGHT} {OUTER} JOIN table_2 ON join_condition_1
    [{{LEFT|RIGHT} {OUTER} JOIN table_3 ON join_condition_2]
    ...
  • The OUTER keyword is optional and typically omitted.

You can use USING keyword to simplify the syntax for joining tables.

You can use NATURAL keyword to create a natural join that joins two tables based on all columns in the two tables that have the same name.

Explicit syntax for a cross join:

SELECT select_list
FROM table_1 CROSS JOIN table_2

Implicit syntax for a cross join:

SELECT select_list
FROM table_1, table_2

A union combines rows from two or more result sets, whilce a join combines columns from two or more result sets.

If you want to sort the result of a union operation, you can code an ORDER BY clause after the last SELECT statement. In an ORDER BY clause, you must use the column names that are specified in the first SELECT statement.

  • That's because the column names in the first SELECT statement are the ones that are used in the final result set.

INSERT, UPDATE and DELETE

Syntax of the CREATE TABLE AS statement:

CREATE TABLE table_name AS select_statement
  • When you use the CREATE TABLE AS statement to create a table, only the column definitions and data are copied.
  • Definitions of primary keys, foreign keys, indexes and so on are not included in the new table.

Syntax of the INSERT statement

INSERT [INTO] table_name [(column_list)]
VALUES (expression_1[, expression_2]...)[,
       (expression_1[, expression_2]...)]...
  • INTO keyword is optional
  • If you don't include a column list in the INSERT clause, you must specify the column values in the same order as in the table, and you must code a value for each column.
  • If you include a column list, you must specify the column values in the same order as in the column list, and you can omit columns that have default values, accept null values, or are automatically generated.
  • To insert a null value into a column, you can use the NULL keyword.
  • To insert a default value or to have MySQL generate a value for an auto increment column, you can use the DEFAULT keyword.

Syntax of the UPDATE statement

UPDATE table_name 
SET column_name_1 = expression_1[, column_name_2 = expression_2]...
[WHERE search_condition]

Syntax of the DELETE statement

DELEtE FROM table_name
[WHERE search_conditon]

Summary Queries

A summary query is a SELECT statement that includes one or more aggregate functions.

All of the aggregate functions except for COUNT(*) ignore null values.

With two exceptions, a SELECT clause that contains an aggregate function can contain only aggregate functions.

  • The first exception is if the column specification results in a literal value.
  • The second exception is if the query includes a GROUP BY clause. Then, the SELECT clause can include any columns specified in the GROUP BY clause.

When a SELECT statement includes a GROUP BY clause, the SELECT clause can include the columns used for grouping, aggregate functions, and expressions that result in a constant value.

When you include a WHERE clause in a SELECT statement that uses grouping and aggregates, MySQL applies the search condition before it groups the rows and calculates the aggregates.

  • A WHERE clause can refer to any column in the base tables.
  • A WHERE clause can't contain aggregate functions.

When you include a HAVING clause in a SELECT statement that uses grouping and aggregates, MySQL applies the search condition after it groups the rows and calculates the aggregates.

  • A HAVING clause can only refer to a column included in the SELECT clause.
  • A HAVING clause can contain aggregate functions.

Subqueries

A subquery is a SELECT statement that's coded within another SQL statement.

  • For this to work, you must enclose the subquery in parentheses.

A subquery can return a single value, a list of values (a result set that has a single column), or a table of values (a result set that has multiple columns).

A subquery can be coded, or introduced, anywhere a single value, a list of values, or a table is allowed.

However, a subquery can't include an ORDER BY clause.

Subqueries can be nested within other subqueries.

When you use the IN operator, the subquery must return a single column of values.

A subquery that's coded in the FROM clause returns a result set that can be referred to as an inline view.

When you code a subquery in the FROM clause, you must assign an alias to it.
